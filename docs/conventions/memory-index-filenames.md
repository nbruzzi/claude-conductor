# Memory-index filenames — a cross-repo convention

The shared memory directory (`~/.claude/projects/<slug>/memory/`) contains
per-fact memo files **and** two machine-maintained index files:

| File             | Role                                                                                                |
| ---------------- | --------------------------------------------------------------------------------------------------- |
| `MEMORY.md`      | The always-loaded **hot subset** (harness-injected at SessionStart).                                |
| `MEMORY-FULL.md` | The **complete** index (read on-demand; generated by the dotfiles `scripts/regen-memory-index.ts`). |

Introduced by the **tiered-memory-index** slice (decouples always-loaded bytes
from total memory count): the dotfiles `regen` writes both files; per-write
index-sync appends to both. See the dotfiles `CLAUDE.md` "Memory Conventions"
for the producer side.

## The invariant (the contract)

Any consumer that (a) `readdirSync`-enumerates the memory dir, (b) classifies a
memory-dir path by basename, or (c) reads `MEMORY.md`'s contents as the corpus
index **must treat BOTH index files as index files** — never score, archive,
parse, or count them as memos.

## Why conductor keeps its OWN `isIndexFile`

The filename set (`MEMORY.md` + `MEMORY-FULL.md`) is a **string convention
shared across two repos** — `claude-conductor` (this plugin) and
`claude-dotfiles` (the substrate) — **not a shared symbol**. The substrate is
not importable here (cross-edge), so conductor pins its own copy:

- **Conductor:** `isIndexFile` + `MEMORY_FILE` / `MEMORY_FULL_FILE` in
  [`src/shared/paths.ts`](../../src/shared/paths.ts), unit-tested in
  `test/shared/paths.test.ts`.
- **Dotfiles:** the leaf `src/shared/memory-index-shared.ts` (same function
  name, same two-filename semantics).

Each repo carries its own implementation **and** its own unit test asserting
both filenames are excluded. That paired discipline — not an import — is the
contract.

### The lesson this encodes (REV-5 census)

The tiered-index design audit found a recurring "missed consumer" class:
three rounds of **dotfiles-scoped** greps missed the conductor consumers because
they keyed on **different literals** (`basename === "MEMORY"` with the extension
stripped; `name === "MEMORY.md"`; `path.endsWith("MEMORY.md")`). A single
exhaustive cross-repo census, keyed on the `MEMORY-FULL.md` artifact, closed the
set. Routing every consumer through `isIndexFile` (rather than a one-off literal)
is what keeps the next index-file addition from re-opening the class.

## Live conductor consumers (excluded via `isIndexFile`)

| Site                                              | Pre-tiered literal                       | Now                                                                                                                                |
| ------------------------------------------------- | ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `src/hooks/checks/memory-attention-updater.ts`    | `basename(filePath, ".md") === "MEMORY"` | `isIndexFile(basename(filePath))` — **highest impact**: a tracked `MEMORY-FULL.md` edit would corrupt the AW1 archival apply-data. |
| `src/memory-attention/cli.ts` (`listMemoryFiles`) | `name === "MEMORY.md"`                   | `isIndexFile(name)`                                                                                                                |
| `src/lexicon/cli.ts` (`scanMemorySource`)         | `path.endsWith("MEMORY.md")`             | `isIndexFile(basename(path))` (exact-basename, not suffix)                                                                         |

`src/memory-loader/index.ts` (`loadMemories`) is **dormant** (zero non-test
callers) and filters `name !== "INDEX.md"` — a third convention that never
matched `MEMORY.md` either, so `MEMORY-FULL.md` is symmetric. It carries a
code-comment note: if ever wired to the real corpus, it must adopt
`isIndexFile`.
