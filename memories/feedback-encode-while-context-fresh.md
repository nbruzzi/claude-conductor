When facing a choice between (a) doing a substantive follow-up — skill update, doc improvement, structural change, lesson capture — in the current session while the work is fresh, vs (b) deferring it to a later session that reloads via memory file: prefer (a) when the follow-up is compact and the current context is genuinely load-bearing for quality.

Reason: a memory file is a frozen snapshot. The current session has the LIVE context — the actual diff, the actual failure mode, the actual surrounding state, the actual conversation that surfaced the lesson. Reloading later via a memory file is reconstruction; encoding now is original capture. Original capture is higher fidelity, and the work itself benefits — for example, an audit-skill update written in the same session that produced the audit-blind-spot lesson encodes the lesson concretely; the same update written from a memory file weeks later encodes it abstractly.

**Why:** The capturing user, on the choice of doing an audit-skill update in the same session as the lesson that motivated it (vs deferring): _"I like this mentality."_ The receipt-in-hand framing is the rule — encode while the substrate, the failure mode, and the fix are all in active context.

**How to apply:**

1. After a substantive landing (PR merge, audit completion, debugging session, hard-won correction): identify any compact follow-ups that would benefit from the current context — skill updates, doc fixes, structural changes that mirror the lesson. Default to doing them in the same session.
2. Compact = under 1 hour. If the follow-up is large (e.g., V1→V2 refactor), defer to a fresh session is fine — but do the lesson capture itself NOW.
3. Frame the trade-off explicitly when sequencing: "Item X benefits from current context, Item Y is fine to defer." Don't bundle them into one offer.
4. Do not generalize to all follow-ups — only ones where the current context is genuinely load-bearing for execution quality. Routine tasks defer fine.
5. The user's explicit approval pattern ("I like this mentality") is the stronger trigger — but absence of approval doesn't mean defer; absence means propose the in-session option and let them decide.
